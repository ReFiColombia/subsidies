Responsive + PWA: Two Approaches

The claim page is already mostly well-structured. Here's what each approach covers.

A

Targeted Fixes + Basic PWA (Recommended)

Fix the 4-5 specific mobile issues + make it installable as a PWA. Minimal changes, maximum impact.

Includes

  • QuickAmountPicker: 2 cols on mobile, 4 on sm+
  • Navbar: tighten spacing for small phones
  • Swap widget overlay: add dark backdrop + scroll lock
  • DonationProgress: stack vertically on small screens
  • PWA manifest.json with app metadata
  • App icons (192x192, 512x512) from existing logo
  • iOS meta tags (apple-mobile-web-app-capable, theme-color)
  • Proper viewport for PWA (viewport-fit=cover)

Skips

  • No responsive card padding changes
  • No safe-area-inset handling for notched phones
  • No splash screen images
B

Polished Mobile + Full PWA

Everything in A, plus extra mobile polish and full PWA experience with splash screens.

Adds on top of A

  • Responsive card padding (p-4 sm:p-6 instead of fixed p-6)
  • Safe-area-inset padding for iPhone notch/Dynamic Island
  • PWA splash screen images for iOS
  • Standalone display with status bar theming
  • Chart tooltip responsive width

Trade-offs

  • More files touched = more review needed
  • Splash screen images need manual creation per device size
  • Safe-area changes could affect desktop layout if not careful
